# 奥特曼亮刀闯任务，梁文锋开卷会昇腾

发布时间：2026-04-27 14:03:21 +0800

本期是「AI 观测站 · 周长文」第一期，固定节奏每周一更。这一期把 GPT-5.5、DeepSeek V4、AI 写代码、GPT Image 2 和国产算力放在同一张图里，回答一个问题：为什么本周 AI 不是单点升级，而是工作流被整体重写。

本期要点

1) 00:00 开场判断：AI 已经进入"下半场"，竞争从"模型分数高一点"升级到"一整套工作流谁能落地"。

2) 00:51 本周三大信号合流：OpenAI 4 月 23 日发布 GPT-5.5，先进 ChatGPT 付费入口和 Codex；同周 ChatGPT Images 2.0 上线，API 模型名 gpt-image-2；DeepSeek 4 月 24 日发布 V4 预览（V4-Pro / V4-Flash），据 Reuters 报道华为 Ascend supernode 将支持 V4。

3) 02:31 不再只看跑分：模型从"产生内容"走向"完成工作"——保持上下文、调用工具、生成中间结果、等待人类验收，人的位置同时往前移和往后移。

4) 04:44 老办法不够用了：丢掉"看参数、看榜单"，建立四个新问题——能否执行长任务、能否稳定调用工具、成本能否进入真实生产、能否被放进可控的产品和基础设施里。

5) 06:15 GPT-5.5 不是"GPT 又聪明了一点"：能力定位是 agentic coding、计算机操作、知识工作、早期科研；OpenAI 把它放进"干活"的位置，先进入两个典型工作场景——知识劳动（ChatGPT）和代码生产（Codex）。

6) 08:12 GPT-5.5 暴露的边界：API Pricing 页标 coming soon，发布当天没有全量进 API；System Card 写明预部署安全评估、红队、Preparedness Framework、test-time compute——能力越靠近真实执行，分层访问越成为产品本身。

7) 11:13 DeepSeek V4 预览版引入：MoE、100 万 token 上下文、reasoning 与 agentic coding 能力定位；API 文档已列 V4-Pro / V4-Flash 价格与最高 384K 输出。同时讲清边界：不能说 V4 全面超过 GPT/Claude/Gemini；不能说 V4 完全由华为昇腾训练。

8) 12:53 V4 真正的组合拳：长上下文 + Pro/Flash 分层 + API 价格 + 开源生态 + 国产算力适配信号——把企业关心的"用得起、接得上、跑得稳"放进同一张图里，让 V4 从"模型论文"变成"可进入产业系统的组件"。

9) 15:09 两条路线的差异：OpenAI 把模型能力深度绑定 ChatGPT、Codex 等产品入口；DeepSeek 把开源、长上下文、低价 API 与国产算力适配捆在一起卖给开发者和企业——竞争已经从"模型 vs 模型"变成"工作流入口 vs 工作流入口"。

10) 18:28 下半场判断框架（核心交付）：① 能不能执行长任务；② 能不能稳定调用工具；③ 成本能不能进入真实生产；④ 能不能被放进可控的产品和基础设施里。这四问以后每周都会拿出来检验新模型、新发布、新工具。

11) 19:08 把本周套进框架：GPT-5.5 推第一问与第四问（执行能力 + 安全分层访问）；DeepSeek V4 补第二问与第三问（长上下文+工具调用、价格+国产算力）。

12) 19:42 接下来盯什么：AI 写代码（coding agent 是最早爆发的工作流入口）、图像生产（GPT Image 2 已经在把封面、海报、广告、设计稿往生产线推）。我们之后接着讲。
